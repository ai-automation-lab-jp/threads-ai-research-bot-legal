データ削除案内

本ツールは、削除依頼を自動処理するコールバックAPIや、削除状況を確認できるWebページ・APIを実装していません。本ページに記載する手順はすべて、運営者による手動対応です。

1. 削除を依頼できるデータ

本ツールがローカルに保存するデータのうち、削除を依頼できるものは以下のとおりです。

  • raw JSON（Threads APIから取得した生データ）
  • processed JSON（分類・スコアリング等の加工後のデータ）
  • 人間向けJSON（processedデータの複製として出力されるもの）
  • CSV出力ファイル
  • Markdownレポート
  • ログ（実行時に出力される情報。標準構成ではログファイルへの永続保存は行いません）
  • アクセストークン（運営者自身の認証情報であり投稿者のデータではありませんが、参考として対象に含みます）

2. 削除依頼方法

ご自身の投稿がこのツールの取得対象に含まれており、データの削除を希望される場合は、下記の連絡先までご連絡ください。

ai.automation.lab.jp@gmail.com

3. 削除依頼時に必要な確認情報

削除を希望する投稿を特定できる情報（投稿URL等、ご自身が公開してよい範囲の情報）をお知らせください。ご依頼の内容によっては、依頼者ご本人からのご連絡であることや、削除対象データを正しく特定するために、運営者から追加の確認をお願いする場合があります。

4. 削除処理の概要

運営者が、削除依頼を受けたのち、上記の確認を行ったうえで、該当する実行単位（run_id）のローカル保存ファイルを手動で特定し、削除します。自動化された削除処理・削除APIは実装していません。

5. 削除完了の連絡方法

削除完了後、依頼時にご利用いただいた連絡先（メールアドレス）宛に、削除完了の旨をメールでご連絡します。

6. GitHubへ実データを保存していないこと

本ツールの実行によって生成されるrawデータ・processedデータ・出力ファイルはGit管理対象外であり、公開リポジトリを含むGitHubリポジトリの履歴に実データが保存されることはありません。アクセストークンについても同様にGit管理対象外であり、不要になったトークンは失効させたうえで運営者の手元から削除します。

7. 自動データ削除コールバックは未実装であること

本ツールはWebサーバー・APIサーバーを持たないCLIツールであり、自動化されたデータ削除コールバックAPI、および削除状況を自動で確認できるWebページ・APIは実装していません。本ページに記載の手順はすべて手動対応です。

8. 問い合わせメール

ai.automation.lab.jp@gmail.com

9. Threads側でのアプリ連携解除について

本ツールとご自身のThreadsアカウントとの連携を停止したい場合は、Threadsアプリまたはウェブ版の設定画面から、連携中のアプリ一覧を確認し、本ツールとの連携を解除することが可能です。具体的な操作方法は、Threads側の仕様変更により変わる場合がありますので、最新の公式案内をご確認ください。

Data Deletion Instructions

This tool does not implement an automated deletion callback API, nor any web page or API for checking deletion status. Every procedure described on this page is handled manually by the operator.

1. Data you can request to be deleted

Among the data this tool stores locally, the following can be requested for deletion.

  • Raw JSON (raw data retrieved from the Threads API)
  • Processed JSON (data after classification and scoring)
  • Human-readable JSON (a duplicate output of processed data)
  • CSV output files
  • Markdown reports
  • Logs (output at runtime; not persisted to a log file in the standard configuration)
  • Access token (the operator's own credential, not a poster's data, but included here for reference)

2. How to request deletion

If your post is included in data collected by this tool and you would like it deleted, please contact us at the address below.

ai.automation.lab.jp@gmail.com

3. Information needed to process a deletion request

Please provide information that identifies the post you want deleted (for example, the post URL), limited to information you are comfortable sharing publicly. Depending on the request, the operator may ask for additional confirmation to verify the requester's identity or to correctly identify the data to be deleted.

4. Overview of the deletion process

After receiving a deletion request and completing the confirmation above, the operator manually identifies the locally stored files for the relevant run (identified by run_id) and deletes them. No automated deletion process or deletion API is implemented.

5. How deletion completion is communicated

After deletion is complete, the operator will send an email to the contact address used to submit the request, confirming that deletion is complete.

6. No real data is stored on GitHub

Raw data, processed data, and output files generated by running this tool are excluded from Git and are never stored in the history of any GitHub repository, including this public one. Access tokens are likewise excluded from Git; tokens that are no longer needed are revoked and removed from the operator's environment.

7. No automated data deletion callback is implemented

This tool is a CLI tool with no web server or API server. No automated data deletion callback API, and no web page or API for checking deletion status, is implemented. Every procedure on this page is a manual process.

8. Contact email

ai.automation.lab.jp@gmail.com

9. Revoking this tool's access from Threads

If you would like to stop this tool's access to your Threads account, you can generally do so from the Threads app or web settings, under the list of connected apps, by revoking access for this tool. The exact steps may change as Threads updates its own settings, so please refer to Threads' current official guidance.
